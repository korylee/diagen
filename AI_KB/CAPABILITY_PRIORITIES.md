# 对标能力优先级

目标：
- 基于 `draw.io / ProcessOn` 的可用性差距，明确 Diagen 后续开发的真实优先级。
- 避免只围绕文件协议与壳层功能推进，而忽略真正决定产品可用性的编辑能力。

## 1. 当前判断

如果以“像真正可用的图编辑器”作为目标，当前最关键的不是：
- page
- 持久化
- 导入导出

而是下面这些直接影响编辑体验的能力：
- 文本编辑
- 连线编辑成熟度
- 容器与层级语义
- 样式体系
- 导航与高频编辑效率

结论：
- `page + 持久化 + 导入导出` 重要，但可以后置。
- 先把编辑器的核心生产力能力补齐，更符合对标 `draw.io / ProcessOn` 的方向。

## 2. P0 优先级

### 文本编辑
需要关注：
- 双击进入编辑态
- Enter / Esc 提交与取消
- 文本框内换行与对齐
- 文本在 shape / linker 上的定位
- 文本编辑与 selection / history 的边界

原因：
- 没有成熟文本编辑，编辑器很难进入真正可用阶段。

### 连线编辑成熟度
需要关注：
- 端点重连
- 控制点编辑
- 正交线手动调整
- 线上的文本定位
- line jump 质量
- 锚点 / 周界吸附一致性

原因：
- `draw.io / ProcessOn` 的核心体验有很大一部分来自连线质量。

### 容器与层级语义
需要关注：
- parent / child 语义
- 容器内拖拽与脱离
- 容器选择与批量操作
- 跨容器移动后的关系修正

原因：
- 只有 group 不足以支撑流程图、组织结构图等真实场景。

## 3. P1 优先级

### 样式体系
需要关注：
- 默认样式
- 批量应用样式
- 线条与填充 preset
- 主题与风格复用

### 导航与高频编辑效率
需要关注：
- space 平移
- minimap
- zoom preset / actual size / fit
- 键盘微移
- 高频快捷键一致性

原因：
- 这些能力会显著影响长时间编辑体验。

## 4. P2 优先级

### 多 Page 分页
决策：
- 进入 `core`
- `Diagram` 直接作为文件根模型
- 分页 UI 在应用层

### 正式持久化
决策：
- 正式文件格式默认围绕 `Diagram`
- 宿主本地快照单独承载 `view / savedAt / activePageId`

### 正式导入导出
决策：
- 默认导入导出整份 `Diagram`
- “导出当前页”是后续衍生功能

原因：
- 这些能力依赖稳定文档模型，适合在核心编辑语义成熟后再做。

## 5. P3 优先级

### 图库与扩展能力
- shape library 分类
- 搜索
- 自定义图元
- 业务 schema 扩展

### 保存体验与工作流
- dirty 状态
- 自动保存提示
- 恢复提示
- 导入覆盖确认

## 6. 当前推荐顺序
1. 文本编辑
2. 连线编辑成熟度
3. 容器与层级语义
4. 样式体系
5. 导航与高频编辑效率
6. 多 page 分页
7. 正式持久化
8. 正式导入导出
9. 保存体验与工作流

## 7. 使用方式
- 若要判断某个功能是否值得提前做，先看它是否直接提升编辑生产力。
- 若某个能力主要服务文件协议、保存或外层壳子，但不直接改善编辑体验，默认后置。
