# 对标能力优先级

目标：
- 基于 `draw.io / ProcessOn` 的可用性差距，明确 Diagen 后续开发的真实优先级。
- 避免只围绕文件协议与壳层功能推进，而忽略真正决定产品可用性的编辑能力。

## 1. 使用说明

这份文档定义的是**长期能力优先级**，不是"当前正在执行的阶段"。

需要按下面两层来理解：
- **长期优先级**：决定哪些能力最终更值得投入，体现对标 `draw.io / ProcessOn` 的真实差距。
- **当前执行阶段**：结合已完成阶段、现有代码基础和返工成本，决定当前迭代先落哪一块。

当前判断：
- 连线编辑成熟度阶段已完成（含端点重连、控制点编辑、正交线路调整、标签定位、line jump、LinkerEndpoint binding 收口）。
- 容器与层级语义阶段已完成。
- 当前执行阶段为"样式体系与导航效率"（Phase4）。
- `page + 持久化 + 导入导出` 依然重要，但继续后置。

## 2. 当前判断

如果以"像真正可用的图编辑器"作为目标，当前最关键的不是：
- page
- 持久化
- 导入导出

而是下面这些直接影响编辑体验的能力：
- 连线编辑成熟度（已完成）
- 容器与层级语义（已完成）
- 样式体系与导航效率（进行中）
- 导航与高频编辑效率（进行中）

结论：
- 文本编辑最小闭环已完成。
- 连线编辑成熟度阶段已完成，LinkerEndpoint binding 模型已收口为 `free | anchor | edge`。
- 容器与层级语义已进入已完成能力集合。
- 当前进行中的第一优先级是"样式体系与导航效率"——导航动作入口基本齐全（缺 actual size），默认样式编辑与批量样式应用待补。
- `page + 持久化 + 导入导出` 重要，但可以后置。

## 3. P0 优先级（已完成）

### 连线编辑成熟度
- 端点重连
- 控制点编辑
- 正交线手动调整
- 线上的文本定位
- line jump 质量
- 锚点 / 周界吸附一致性
- LinkerEndpoint binding 模型收口（`free | anchor | edge`）

### 容器与层级语义
- parent / child 语义
- 容器内拖拽与脱离
- 容器选择与批量操作
- 跨容器移动后的关系修正

## 4. P1 优先级（进行中）

### 样式体系
- 默认样式（Schema 层已具备注册入口，UI 编辑入口待补）
- 批量应用样式（尚未实现）
- 线条与填充 preset
- 主题与风格复用

### 导航与高频编辑效率
- space 平移（已实现）
- minimap（待评估）
- zoom preset / actual size / fit（fit / fit-selection 已实现，actual size 待补）
- 键盘微移
- 高频快捷键一致性

## 5. P2 优先级

### 多 Page 分页
- 进入 `core`
- `Diagram` 直接作为文件根模型
- 分页 UI 在应用层

### 正式持久化
- 正式文件格式默认围绕 `Diagram`
- 宿主本地快照单独承载 `view / savedAt / activePageId`

### 正式导入导出
- 默认导入导出整份 `Diagram`
- "导出当前页"是后续衍生功能

## 6. P3 优先级

### 图库与扩展能力
- shape library 分类
- 搜索
- 自定义图元
- 业务 schema 扩展

### 保存体验与工作流
- dirty 状态
- 自动保存提示
- 恢复提示
- 导入覆盖确认

## 7. 当前推荐顺序
1. 连线编辑成熟度（已完成）
2. 容器与层级语义（已完成）
3. 样式体系与导航效率（进行中：补齐 actual size、默认样式编辑、批量样式应用）
4. 多 page 分页
5. 正式持久化
6. 正式导入导出
7. 保存体验与工作流

## 8. 使用方式
- 若要判断某个功能是否值得提前做，先看它是否直接提升编辑生产力。
- 若某个能力主要服务文件协议、保存或外层壳子，但不直接改善编辑体验，默认后置。
- 若对"当前阶段"有疑问，以 `docs/PROJECT_OVERVIEW.md` 与 `docs/CURRENT_PLAN.md` 为准；本文件只负责长期排序。
